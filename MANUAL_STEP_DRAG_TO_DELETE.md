# 手动步骤拖拽删除功能

## 功能说明

在排班页面，现在可以通过将手动步骤拖动到左侧的"待排班"区域来删除该步骤。

## 使用方法

1. 在排班页面，找到司机行中的手动步骤卡片（灰色左边框）
2. 点击并拖动该手动步骤卡片
3. 将卡片拖到左侧的"待排班"区域
4. 松开鼠标，手动步骤将被删除
5. 系统会自动重新编号后续步骤

## 技术实现

### 修改位置
`src/pages/DispatchPage.tsx` - `onDragEnd` 函数

### 实现逻辑

```typescript
// 在 onDragEnd 函数开始处添加检测
if (activeIdStr.startsWith('step:') && overIdStr === BACKLOG_ID) {
  const stepId = activeIdStr.slice(5);
  const step = currentJobSteps.find(s => s.id === stepId);
  
  if (step && step.node_type === 'step') {
    console.log('手动步骤拖到待排班区域，删除步骤:', stepId);
    deleteManualStep.mutate(stepId);
    return;
  }
}
```

### 删除步骤的处理

`deleteManualStep` mutation 会：
1. 从数据库删除该步骤
2. 自动更新后续步骤的 `step_number`，保持连续性
3. 刷新页面数据

## 交互细节

### 可以拖到待排班区域删除的：
- ✅ 手动步骤（`node_type === 'step'`）

### 不能拖到待排班区域删除的：
- ❌ 订单节点（`node_type === 'order'`）
  - 订单节点拖到待排班区域会取消分配，但不会删除订单本身
  - 订单仍然会出现在待排班列表中

### 其他拖拽行为保持不变：
- 司机行内拖拽排序：订单和手动步骤都可以在同一司机行内重新排序
- 订单从待排班拖到司机行：分配订单给司机
- 订单从司机行拖回待排班：取消分配
- 订单跨司机拖动：重新分配给其他司机

## 用户体验

### 优点
1. **直观**：拖到待排班区域 = 删除，符合用户直觉
2. **快速**：无需点击菜单或确认对话框
3. **一致**：与订单的拖拽行为保持一致的交互模式

### 安全性
- 删除操作立即执行，无确认对话框
- 如果误删，需要重新添加手动步骤
- 建议：如果需要更安全的操作，可以添加撤销功能或确认对话框

## 视觉反馈

当拖动手动步骤到待排班区域时：
- 待排班区域会高亮显示（通过 `useDroppable` 的 `isOver` 状态）
- 松开鼠标后，步骤立即消失
- 显示 toast 提示："已删除步骤"

## 测试建议

1. **基本删除**：
   - 添加一个手动步骤
   - 拖到待排班区域
   - 验证步骤被删除

2. **步骤重新编号**：
   - 添加多个步骤（例如：步骤1、2、3）
   - 删除中间的步骤（步骤2）
   - 验证步骤3自动变成步骤2

3. **订单节点不受影响**：
   - 尝试拖动订单节点到待排班区域
   - 验证订单只是取消分配，不会被删除
   - 验证订单仍在待排班列表中

4. **混合场景**：
   - 创建包含订单和手动步骤的司机行
   - 删除手动步骤
   - 验证订单节点不受影响
   - 验证所有步骤的编号正确
