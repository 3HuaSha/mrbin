# 手动步骤系统实现说明

## 概述
已成功实现排班页面的手动步骤系统，支持调度员在订单节点之间插入自定义步骤（如"回基地放桶"、"去料场装料"等）。

## 数据库变更

### 1. 新增 `common_locations` 表
存储常用地点，方便调度员快速选择：
- Kennedy Depot (3445 Kennedy Rd)
- Sheppard Yard (12441 Sheppard Ave)

### 2. 更新 `job_steps` 表
添加以下字段以支持两种节点类型：

**新增字段：**
- `driver_id` (UUID, 必填) - 属于哪个司机
- `scheduled_date` (DATE, 必填) - 哪天执行
- `order_id` (UUID, 可选) - 订单节点才有
- `node_type` (TEXT, 'order' | 'step') - 节点类型标识
- `notes` (TEXT, 可选) - 备注信息
- `bin_id` (UUID, 可选) - 桶号

**字段调整：**
- `assignment_id` - 改为可选（步骤节点不需要）
- `location` - 改为可选

## 前端实现

### 1. 排班页面 (`src/pages/DispatchPage.tsx`)

#### 新增组件：

**OrderNodeDisplay** - 订单节点显示
- 蓝色左边框
- 显示订单信息：类型、桶类型、尺寸、地址、时段、备注
- 支持取消分配

**StepNodeDisplay** - 步骤节点显示
- 灰色左边框
- 显示步骤信息：动作类型、地点、桶号、备注
- 支持删除

**InsertStepButton** - 插入步骤按钮
- 平时显示为细虚线 + 号
- hover 时展开显示完整按钮
- 点击后展开 inline 面板，包含：
  - 地点下拉（常用地点 + 手动输入）
  - 动作下拉（取桶、放桶、倒垃圾、装料、卸料）
  - 桶号输入（可选）
  - 备注输入（可选）

#### 更新的组件：

**DriverColumn** - 司机列
- 合并显示订单节点和步骤节点
- 按 `step_number` 排序
- 在每个节点之间显示插入按钮
- 支持拖拽调整顺序（待实现）

#### 新增 Mutations：

**insertManualStep** - 插入手动步骤
- 在指定位置插入新步骤
- 自动重新编号后续步骤

**deleteManualStep** - 删除手动步骤
- 删除指定步骤
- 自动重新编号后续步骤

### 2. 司机端 (`src/pages/driver/DriverHomePage.tsx`)

#### 主要变更：

**查询方式改变：**
- 直接查询 `job_steps` 表，不再通过 `dispatch_assignments`
- 按 `step_number` 顺序显示所有步骤

**步骤锁定逻辑：**
- 只有前一个步骤完成才能解锁下一步
- 使用 `useMemo` 计算每个步骤的锁定状态
- 锁定的步骤显示 🔒 图标和"请先完成上一步"提示

**节点类型区分：**
- 订单节点：显示订单号、客户信息、桶类型等
- 步骤节点：显示"手动步骤"标签、动作类型、地点等

## 使用流程

### 调度员操作：

1. **查看司机任务列表**
   - 每个司机显示所有节点（订单 + 步骤）
   - 按执行顺序排列

2. **插入手动步骤**
   - 点击节点之间的 "+ 插入步骤" 按钮
   - 选择地点（常用地点或手动输入）
   - 选择动作类型
   - 可选填写桶号和备注
   - 点击"确认插入"

3. **管理步骤**
   - 订单节点：可拖回左边取消分配
   - 步骤节点：可编辑或删除
   - 支持拖拽调整顺序（待实现）

### 司机操作：

1. **查看任务列表**
   - 看到所有步骤（订单 + 手动步骤）
   - 按顺序显示，完成当前步骤才能进行下一步

2. **执行步骤**
   - 只能执行未锁定的步骤
   - 完成后自动解锁下一步

## 技术细节

### 数据流：

```
调度员插入步骤
  ↓
insertManualStep mutation
  ↓
写入 job_steps 表 (node_type='step')
  ↓
更新后续步骤编号
  ↓
刷新查询
  ↓
司机端看到新步骤
```

### 步骤编号规则：

- 每个司机每天的步骤从 1 开始连续编号
- 插入步骤时，后续步骤编号 +1
- 删除步骤时，后续步骤编号 -1

### 节点类型区分：

- `node_type='order'`: 订单节点，有 `order_id` 和 `assignment_id`
- `node_type='step'`: 手动步骤，没有 `order_id` 和 `assignment_id`
