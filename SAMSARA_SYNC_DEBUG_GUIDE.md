# Samsara 同步调试指南

## 🔍 如何查看详细的同步日志

我已经在代码中添加了非常详细的调试日志，可以帮你找出为什么某些车辆（如 FLAT#3）没有被同步。

### 方法 1: 查看浏览器控制台（前端日志）

1. **打开车队管理页面**
   - 访问你的应用 URL
   - 进入"司机与车辆"页面

2. **打开浏览器开发者工具**
   - 按 `F12` 或右键点击页面 → "检查"
   - 切换到 **Console（控制台）** 标签

3. **点击"从 Samsara 同步"按钮**

4. **查看控制台输出**，你会看到类似这样的日志：
   ```

   📦 前端: 收到 Samsara 响应: {...}

     1. BIN#1 (ID: 12345678...)
     2. BIN#2 (ID: 23456789...)
     3. FLAT#1 (ID: 34567890...)
     ...
   🔄 正在清理旧的派遣数据...
   ✅ 已完成深度清理
     ➕ 准备插入: BIN#1 → 类型=HINO, 容量=20yd
     ➕ 准备插入: FLAT#1 → 类型=MACK, 容量=40yd
     ...

   ✅ 车辆插入成功
   ✅ 前端: 同步完成，刷新车辆列表
   ```

### 方法 2: 查看 Railway 服务端日志（后端日志）

1. **登录 Railway Dashboard**
   - 访问 https://railway.app
   - 进入你的项目

2. **查看部署日志**
   - 点击你的服务
   - 切换到 **Deployments** 标签
   - 点击最新的部署
   - 查看 **Logs（日志）**

## 🎯 关键信息

### 查找 FLAT#3 的线索

在日志中搜索以下内容：

1. **搜索 "FLAT#3"**
   - 看看它是否出现在任何 API 响应中
   - 确认它来自哪个端点（Vehicles, Trailers, Equipment, Assets）

2. **检查分页信息**
   - 看看 `hasNextPage` 是否为 `true`
   - 如果是，检查是否正确获取了下一页

3. **检查去重过程**
   - 看看 FLAT#3 是否被标记为"跳过重复单位"
   - 确认它是否成功添加到最终列表

4. **检查最终合并列表**
   - 确认 FLAT#3 是否在最终的 15 个单位中

## 🐛 常见问题

### 问题 1: 车辆在 Trailers 端点而不是 Vehicles 端点
**症状**: 只看到 10 辆车，但实际有更多
**原因**: FLAT 可能被 Samsara 归类为 Trailer（拖车）而不是 Vehicle
**解决**: 代码已经查询了 6 个不同的端点，应该能获取到所有类型

### 问题 2: 分页没有正确处理
**症状**: 日志显示 `hasNextPage=true` 但没有获取下一页
**原因**: 分页逻辑有问题
**解决**: 检查 `endCursor` 是否正确传递

### 问题 3: 车辆被去重过滤掉
**症状**: 车辆出现在原始数据中，但不在最终列表中
**原因**: 可能有重复的 ID 或名称
**解决**: 查看"跳过重复单位"的日志

### 问题 4: API 权限不足
**症状**: 某些端点返回 403 或 401 错误
**原因**: Samsara API Token 没有足够的权限
**解决**: 检查 API Token 的权限设置
